# Book-Derived Essence Index

This index records the debating pass used to reduce skill homogeneity. It is a repo-level guide for future Darwin evolution: preserve each skill's native framework language, discovery method, and adjacent-skill boundary instead of regressing to generic workflow advice.

## Editing Rule

For every skill, the first runtime lens must be its book-derived kernel:

- Core framework: the native objects and relations from the source.
- Deep idea: the non-obvious source claim that makes the skill worth invoking.
- Discovery method: how the theory finds hidden structure, failure, ambiguity, or leverage.
- Boundary: when this theory should downgrade or hand off to a neighboring skill.

Portable closure remains mandatory, but it belongs in source closure and references, not as the main intellectual content.

## Representative Source Kernels

- Engineering cybernetics: classify system dynamics before choosing stability, delay, noise, noninteraction, optimizing, or redundancy methods.
- Category theory fusion: use ologs, path equivalence, functors, pullbacks, pushouts, monads, and operads only when structure preservation matters.
- Sowa conceptual graphs: preserve logical commitments across human graph notation, CGIF/Common Logic interchange, and predicate-style mappings.
- Sowa KR: every representation must answer logic, ontology, and computation together.
- Sowa ontology: categories are commitments for a language and purpose, not decorative labels.
- SRE: error budgets turn reliability into an explicit product tradeoff.
- Process mining: the real process is reconstructed from event traces, not assumed from diagrams.
- Model checking: the counterexample trace is a first-class discovery artifact.
- Requisite variety: disturbance variety and response variety must be balanced before adding automation.

## Darwin Evolution Checks

Future evolution should reject a candidate update when it:

- Replaces native theory terms with generic planning terms.
- Removes discovery methods and leaves only output templates.
- Uses portable closure language as the dominant content.
- Fails to distinguish adjacent skills listed above.
- Cannot name the source kernel and at least three native artifacts or checks for the skill.
