# Rejected Or Merged Units

## R1. Lambda Calculus As A Standalone Skill

Decision: rejected as standalone, merged lightly into the function/intension background.

Reason:

Lambda calculus is important in the source, but the requested focus is knowledge modeling through sets, relations, graphs, logic, model theory, truth semantics, and controlled English. A full lambda-calculus execution skill would dilute the modeling workflow.

## R2. Game Playing As AI Search Skill

Decision: rejected as standalone, retained only as support for game-theoretic semantics.

Reason:

The game graph section explains minimax and backed-up values, but the output skill is not meant for chess/game AI. Its useful modeling role is the proposer/skeptic evaluation of quantified formulas.

## R3. Formal Grammar Construction As Parser Skill

Decision: rejected as standalone, merged into controlled-English syntax checks.

Reason:

The source covers formal grammars in detail, but the requested skill should not become a grammar engineering skill. The retained part is the need to define accepted sentence forms before semantic translation.

## R4. Philosophical Defense Of Correspondence Truth

Decision: rejected.

Reason:

Tarski explicitly avoids settling the "right" conception of truth. The executable takeaway is material adequacy, satisfaction, language hierarchy, and the separation of truth from proof.

## R5. General Math Homework Tutor

Decision: rejected.

Reason:

The book includes introductory examples for sets, functions, graphs, and logic. The skill is intended for Codex knowledge-modeling work, not step-by-step school exercises.

## R6. Complete Theorem Prover Procedure

Decision: rejected.

Reason:

The source introduces proof theory and inference rules but does not provide an implementation-ready theorem prover. The skill keeps proof as an audit status and validation technique.
