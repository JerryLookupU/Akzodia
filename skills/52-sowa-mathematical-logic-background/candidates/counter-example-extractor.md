# Candidate Counterexamples And Failure Patterns

## CE1. COBOL-Style Fixed Ontology

Failure:

A language can be readable but locked to a computer-oriented ontology that cannot represent application-domain entities.

Retained lesson:

Controlled English for knowledge modeling should keep content words extensible and domain-defined.

## CE2. Ordinary English Implicature

Failure:

Interpreting `Bob picked up the cup and drank the coffee` as equivalent to the reversed sentence ignores temporal implicature in ordinary English.

Retained lesson:

Controlled English must either formalize such implications or exclude them.

## CE3. Diagram-As-Definition

Failure:

Assuming visual features of a graph drawing have formal meaning when only node and arc sets are defined.

Retained lesson:

Treat diagrams as intuition unless formal definitions mention the visual property.

## CE4. Tree-Only Taxonomy

Failure:

Treating every hierarchy as a tree hides partial orders, multiple inheritance, and lattice operations.

Retained lesson:

Subtype systems often need DAGs or lattices.

## CE5. Semantically Closed Truth Language

Failure:

A language that contains its own sentence names, its own truth predicate, and ordinary logic can reconstruct liar-style contradiction.

Retained lesson:

Separate object language from metalanguage.

## CE6. Truth Equals Provability

Failure:

Assuming every true sentence in a sufficiently rich formal system is provable.

Retained lesson:

Track truth-in-model and provability as different statuses.
