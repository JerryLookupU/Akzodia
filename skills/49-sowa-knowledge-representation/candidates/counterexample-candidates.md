# Counterexample Candidates

## Accepted Warning Patterns

1. Logic without ontology.
   - Evidence: preface says terms and symbols become ill-defined without ontology.
   - Skill failure mode: formal rules over vague domain terms.

2. Ontology without logic.
   - Evidence: preface says without logic there are no criteria for redundancy or contradiction.
   - Skill failure mode: vocabulary theater.

3. Logic and ontology without computation.
   - Evidence: preface says computable models distinguish KR from pure philosophy.
   - Skill failure mode: elegant model that cannot be queried, maintained, or run.

4. Static ontology for dynamic domains.
   - Evidence: chapter 4 and index entries for times, events, processes, Petri nets, situation calculus.
   - Skill failure mode: actions and histories hidden as attributes.

5. Global facts where context matters.
   - Evidence: chapter 5 and context index entries.
   - Skill failure mode: user, source, time, jurisdiction, or agent beliefs leak into universal truth.

6. Monotonic rules for default-heavy reality.
   - Evidence: chapter 6 and index entries for default logic, nonmonotonic logic, open/closed world assumptions.
   - Skill failure mode: conclusions cannot retract when new evidence arrives.

7. Representation translation without semantic audit.
   - Evidence: chapter 7 on relating different representations and multiple paradigms.
   - Skill failure mode: schema adapters appear to work while losing meaning.

8. Source retrieval without knowledge indexes.
   - Evidence: adapted from source emphasis on indexes, ontology, logic, and computable models.
   - Skill failure mode: modern vector system cites text but cannot explain concepts, rules, or revision conditions.
