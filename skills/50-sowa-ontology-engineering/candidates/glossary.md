# Candidate Glossary

## Core Terms Promoted To Skill

- `Ontology`: categories for a language and purpose; not a mere term list.
- `Independent`: entity characterized without needing a relation to another entity.
- `Relative`: entity considered in relation to another entity.
- `Mediating`: entity that brings other entities into relation.
- `Physical`: entity located in space-time.
- `Abstract`: information independent of a particular encoding.
- `Continuant`: entity whose identity can be recognized over time.
- `Occurrent`: entity unfolding in time with temporal parts or stages.
- `Object`: independent physical continuant.
- `Process`: independent physical occurrent.
- `Schema`: independent abstract continuant.
- `Script`: independent abstract occurrent.
- `Description`: abstract proposition about a continuant.
- `History`: abstract proposition about an occurrent.
- `Situation`: mediating physical occurrent.
- `Reason`: mediating abstract continuant.
- `Purpose`: mediating abstract occurrent.
- `Lexicon`: bridge between a language and the knowledge expressed in it.
- `Sign`: entity representing another entity to an agent.
- `Interpretant`: concept or developed sign produced in interpretation.
- `Ground`: respect or capacity under which a sign represents an object.
- `Agent`: animate or artificial entity capable of doing something on purpose.
- `Role`: entity considered in relation to something else.
- `Has`: primitive relation used to define role relations.
- `Participant`: spatial part of an occurrent or role in a process.
- `Stage`: temporal part of an occurrent.
- `Attribute`: property of a continuant.
- `Manner`: property of an occurrent.
- `Initiator`: determinant source participant.
- `Resource`: immanent source participant.
- `Goal`: determinant product participant.
- `Essence`: immanent product participant.
- `State`: condition that may persist for a duration in a discrete process.
- `Event`: change between states in a discrete process.
- `Causal influence`: directed dependency from one state/event node to another.
- `Precondition`: input state type for an event type.
- `Postcondition`: output state type for an event type.
- `Theory lattice`: partial ordering of theories by generalization and specialization.

## Terms Kept As Supporting Concepts

- `FCA`: useful for type hierarchy construction, but too specialized for the final workflow.
- `Conceptual graph`: important notation, but the skill focuses on modeling decisions independent of notation.
- `Petri net`: useful process notation, included as a possible representation rather than a required method.
- `PSL`: relevant standard discussed by Sowa, but not required for the general Codex skill.
