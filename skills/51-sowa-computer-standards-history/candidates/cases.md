# Candidate Cases

## C1: IBM Future Systems / GRAD

Evidence:

- Intended to replace System/370 as new hardware and software architecture.
- Warning signs were visible by 1971; by 1974 people feared public career-killing questions.
- VANDERBILT native FS mode was slower than the System/370 emulator path.
- IBM later reused VANDERBILT circuitry for System/370-compatible machines.

Skill use:

- Primary example for future-system failure, migration risk, and revolutionary replacement.

Status: accepted.

## C2: HLS / AFS / Lost Opportunities

Evidence:

- HLS saw a need for higher-level systems but failed to address smooth migration from System/370.
- AFS and Cocke's RISC path could have supported wide hardware/software tradeoffs.
- One bit per 64-bit word for protected descriptors is presented as a low-change path.

Skill use:

- Example of ambitious architecture that could have been evolutionary instead of revolutionary.

Status: accepted.

## C3: Memo 125 Primary Architecture

Evidence:

- Operating-system primary architecture is a closed list of fundamental interfaces.
- Secondary components can be distributed only after primary architecture is firm.
- COLBY had useful components but no substitute for architecture.

Skill use:

- Core decision framework for platform and API reviews.

Status: accepted.

## C4: PL/I, Algol 68, Ada, OS/2, OSI

Evidence:

- Official standards or major organization-backed systems were displaced by simpler practical systems.

Skill use:

- Evidence set for de facto standard prediction.

Status: accepted.

## C5: Linux API / POSIX-like Portability

Evidence:

- Sowa predicted Linux API would become the de facto standard for operating systems.
- Index notes later Linux dominance across Android, Chrome systems, supercomputers, and server farms.

Skill use:

- Example of using the law predictively, not only retrospectively.

Status: accepted with caution. Prediction framing is useful; exact market claims should be checked if used in current factual analysis.

## C6: Task-Force Tim

Evidence:

- Frequent reorganizations and critical problems each assigned to ad hoc task forces signaled FS trouble.

Skill use:

- Organizational smell for missing architecture and failing integration.

Status: accepted.
