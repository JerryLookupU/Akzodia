# Glossary

- Advanced Future Systems (AFS): IBM project preceding and influencing FS; in Sowa's account, a more flexible and compatible architectural direction.
- COLBY: FS operating-system component name in Memo 125.
- De facto standard: the system or interface adopted in practice, regardless of official status.
- Future Systems (FS): IBM's 1970s program to replace System/370 with a new hardware and software architecture.
- GRAD: code name for IBM's Future Systems project in Memo 125.
- HLS: Higher Level System; study group/report on a higher-level architecture.
- Interim product: a compatible near-term product that adds value while moving users toward the future architecture.
- Law of Standards: Sowa's hypothesis that proactive official standards for X often result in simpler de facto standards for X.
- One-level store: a virtual memory/addressing concept that removes the user-visible distinction between RAM and disk files.
- Primary architecture: the necessary and sufficient core interfaces all programs and secondary facilities depend on.
- RIPON: FS hardware architecture component name.
- Secondary architecture: open-ended facilities such as editors, commands, compilers, query tools, and presentation layers.
- Smooth migration path: a product and architecture plan that lets users move from old to new with little disruption.
- Slogan: a persuasive phrase that must be converted into implementable principles before it can guide design.
- System/370: IBM legacy base whose compatibility and customer ecosystem shaped the FS problem.
- Task force symptom: repeated ad hoc committees and reorganizations used to address architectural crisis symptoms.
- Upward compatibility: ability for existing programs or users to move forward without prohibitively expensive conversion.
